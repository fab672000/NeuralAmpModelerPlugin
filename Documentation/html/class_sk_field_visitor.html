Detailed Description

SkFieldVisitor is an interface that can be implemented by any class to visit all fields of SkReflected types, and of types that implement the visitFields() function.

Classes implementing the interface must supply implementations of virtual functions that visit basic types (float, int, bool, SkString), as well as helper methods for entering the scope of an object or array.

All visit functions supply a field name, and a non-constant reference to an actual field. This allows visitors to serialize or deserialize collections of objects, or perform edits on existing objects.

Classes that implement visitFields (typically derived from SkReflected) should simply call visit() for each of their fields, passing a (unique) field name, and the actual field. If your class has derived fields, it's best to only visit() the fields that you would serialize, then enforce any constraints afterwards.

See SkParticleSerialization.h for example visitors that perform serialization to and from JSON.
