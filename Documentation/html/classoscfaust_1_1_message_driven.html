Detailed Description

a base class for objects accepting OSC messages

Message driven objects are hierarchically organized in a tree. They provides the necessary to dispatch an OSC message to its destination node, according to the message OSC address.

The principle of the dispatch is the following:

  • first the processMessage() method should be called on the top level node
  • next processMessage call propose

handler for the 'get' message

Parameters
ipdestthe output message destination IP

The 'get' message is supported by every node:

  • it is propagated to the subnodes until it reaches terminal nodes
  • a terminal node send its state on 'get' request to the IP address given as parameter. The get method is basically called by the accept method.

virtual

propose an OSc message at a given hierarchy level.

Parameters
msgthe osc message currently processed
regexpa regular expression based on the osc address head
addrTailthe osc address tail

The method first tries to match the regular expression with the object name. When it matches:

  • it calls accept when addrTail is empty
  • or it propose the message to its subnodes when addrTail is not empty. In this case a new regexp is computed with the head of addrTail and a new addrTail as well.
