8 **
9 ** Usage:
10 ** 1. Create a JNL_HTTPGet object, optionally specifying a JNL_AsyncDNS
11 ** object to use (or NULL for none, or JNL_CONNECTION_AUTODNS for auto),
12 ** and the receive buffer size, and a string specifying proxy (or NULL
13 ** for none). See note on proxy string below.
14 ** 2. call addheader() to add whatever headers you want. It is recommended to
15 ** add at least the following two:
16 ** addheader("User-Agent:MyApp (Mozilla)");
17 */// addheader("Accept:*/*");
18 /* ( the comment weirdness is there so I Can do the star-slash :)
19 ** 3. Call connect() with the URL you wish to GET (see URL string note below)
20 ** 4. Call run() once in a while, checking to see if it returns -1
21 ** (if it does return -1, call geterrorstr() to see what the error is).
22 ** (if it returns 1, no big deal, the connection has closed).
23 ** 5. While you're at it, you can call bytes_available() to see if any data
24 ** from the http stream is available, or getheader() to see if any headers
25 ** are available, or getreply() to see the HTTP reply, or getallheaders()
26 ** to get a double null terminated, null delimited list of headers returned.
27 ** 6. If you want to read from the stream, call get_bytes (which returns how much
28 ** was actually read).
29 ** 7. content_length() is a helper function that uses getheader() to check the
30 ** content-length header.
31 ** 8. Delete ye' ol' object when done.
32 **
33 ** Proxy String:
34 ** should be in the format of host:port, or user@host:port, or
35 ** user:password@host:port. if port is not specified, 80 is assumed.
36 ** URL String:
37 ** should be in the format of http://user:pass@host:port/requestwhatever
38 ** note that user, pass, port, and /requestwhatever are all optional :)
39 ** note that also, http:// is really not important. if you do poo://
40 ** or even leave out the http:// altogether, it will still work.
41 */
