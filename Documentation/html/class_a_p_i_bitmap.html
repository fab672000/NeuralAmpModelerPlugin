Detailed Description

A base class interface for a bitmap abstraction around the different drawing back end bitmap representations. In most cases it does own the bitmap data, the exception being with NanoVG, where the image is loaded onto the GPU as a texture, but still needs to be freed. Most of the time end-users will deal with IBitmap rather than APIBitmap, which is used behind the scenes.
