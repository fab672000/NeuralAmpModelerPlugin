\doxysubsection{Detailed Description}
Defines the taper conversion behavior for a parameter. 

\textbackslash{}sdkinternal

This interface represents a delegate class to be used in conjunction with \mbox{\hyperlink{class_a_a_x___i_parameter}{A\+A\+X\+\_\+\+I\+Parameter}}. \mbox{\hyperlink{class_a_a_x___i_parameter}{A\+A\+X\+\_\+\+I\+Parameter}} delegates all conversion operations between normalized and real parameter values to classes that meet this interface. You can think of \mbox{\hyperlink{class_a_a_x___i_taper_delegate}{A\+A\+X\+\_\+\+I\+Taper\+Delegate}} subclasses as simple taper conversion routines that enable a specific taper or range conversion function on an arbitrary parameter.

To demonstrate the use of this interface, we will examine a simple call routine into a parameter\+:

\begin{DoxyParagraph}{}

\begin{DoxyEnumerate}
\item The host application calls into the plug-\/in\textquotesingle{}s \mbox{\hyperlink{class_a_a_x___c_parameter_manager}{A\+A\+X\+\_\+\+C\+Parameter\+Manager}} with a Parameter ID and a new normalized parameter value. This new value could be coming from an automation lane, a control surface, or any other parameter control; from the plug-\/in\textquotesingle{}s perspective, these are all identical.
\end{DoxyEnumerate}
\end{DoxyParagraph}
\begin{DoxyParagraph}{}

\begin{DoxyEnumerate}
\item The \mbox{\hyperlink{class_a_a_x___c_parameter_manager}{A\+A\+X\+\_\+\+C\+Parameter\+Manager}} finds the specified \mbox{\hyperlink{class_a_a_x___c_parameter}{A\+A\+X\+\_\+\+C\+Parameter}} and calls \mbox{\hyperlink{class_a_a_x___i_parameter_ad4b9a53e4e972d1a840ef094036e5a70}{A\+A\+X\+\_\+\+I\+Parameter\+::\+Set\+Normalized\+Value()}} on that parameter
\end{DoxyEnumerate}
\end{DoxyParagraph}
\begin{DoxyParagraph}{}

\begin{DoxyEnumerate}
\item \mbox{\hyperlink{class_a_a_x___i_parameter_ad4b9a53e4e972d1a840ef094036e5a70}{A\+A\+X\+\_\+\+I\+Parameter\+::\+Set\+Normalized\+Value()}} results in a call into the parameter\textquotesingle{}s concrete taper delegate to convert the normalized value to a real value.
\end{DoxyEnumerate}
\end{DoxyParagraph}
Using this pattern, the parameter manager is able to use real parameter values without actually knowing how to perform the conversion between normalized and real values.

The inverse of the above example can also happen, e.\+g. when a control is updated from within the data model. In this case, the parameter can call into its concrete taper delegate in order to normalize the updated value, which can then be passed on to any observers that require normalized values, such as the host app.

For more information about the parameter manager, see the A\+A\+X\+Library\+Features\+\_\+\+Parameter\+Manager documentation page.
