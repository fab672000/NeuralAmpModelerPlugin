This directory contains the Oboe unit tests. They are run using the bash script {\ttfamily run\+\_\+tests.\+sh}.

The basic operation is\+:


\begin{DoxyEnumerate}
\item Connect an Android device or start the Android emulator
\item Open a terminal window and execute {\ttfamily run\+\_\+tests.\+sh}
\end{DoxyEnumerate}\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_ead9722cb175d4272a242dbba8c9e265_autotoc_md514}{}\doxysection{Prerequisites/caveats}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_ead9722cb175d4272a242dbba8c9e265_autotoc_md514}
You must have compiled and executed one of the Oboe examples or Oboe\+Tester. That ensures that the N\+DK and cmake is installed.

You must define {\ttfamily A\+N\+D\+R\+O\+I\+D\+\_\+\+N\+DK} as an environment variable and make sure {\ttfamily cmake} is on your path.

To test this on Mac or Linux enter\+: \begin{DoxyVerb}echo $ANDROID_HOME
echo $ANDROID_NDK
cmake --version
\end{DoxyVerb}


They may already be set. If not, then this may work on Mac OS\+: \begin{DoxyVerb}export ANDROID_HOME=$HOME/Library/Android/sdk
export ANDROID_NDK=$ANDROID_HOME/ndk-bundle
\end{DoxyVerb}


or this may work on Linux\+: \begin{DoxyVerb}export ANDROID_HOME=$HOME/Android/Sdk
ls $ANDROID_HOME/ndk
\end{DoxyVerb}


Make note of the folder name. Mine was \char`\"{}21.\+0.\+6113669\char`\"{} so \mbox{\hyperlink{struct_i}{I}} entered\+: \begin{DoxyVerb}export ANDROID_NDK=$ANDROID_HOME/ndk/21.0.6113669/
\end{DoxyVerb}


If you need to add {\ttfamily cmake} to your path then you can find it by entering\+: \begin{DoxyVerb}ls $ANDROID_HOME/cmake
\end{DoxyVerb}


Make note of the folder name. Mine was \char`\"{}3.\+6.\+4111459\char`\"{} so \mbox{\hyperlink{struct_i}{I}} entered\+: \begin{DoxyVerb}export PATH=$PATH:$ANDROID_HOME/cmake/3.6.4111459/bin
