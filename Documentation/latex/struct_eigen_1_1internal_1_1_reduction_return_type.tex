\subsubsection*{template$<$typename Op, typename Coeff\+Return\+Type$>$\newline
struct Eigen\+::internal\+::\+Reduction\+Return\+Type$<$ Op, Coeff\+Return\+Type $>$}

For S\+Y\+CL, the return type of the reduction is deduced from the initialize method of the given Op. This allows the reduction to have a different type for the accumulator than the input data type. If this is the case, the functor needs to have two reduce method\+: one for reducing an element of the input with the accumulator and the other for reducing two accumulators. Such a reducer can be useful for instance when the accumulator is a boolean or a bitset that checks for some properties of the input.
