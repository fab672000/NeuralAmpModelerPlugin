\doxysubsection{Detailed Description}
Pseudo expression providing broadcasting and partial reduction operations. 


\begin{DoxyTemplParams}{Template Parameters}
{\em Expression\+Type} & the type of the object on which to do partial reductions \\
\hline
{\em Direction} & indicates whether to operate on columns (\mbox{\hyperlink{group__enums_ggad49a7b3738e273eb00932271b36127f7addca718e0564723df21d61b94b1198be}{Vertical}}) or rows (\mbox{\hyperlink{group__enums_ggad49a7b3738e273eb00932271b36127f7aae8a16b3b9272683c1162915f6d892be}{Horizontal}})\\
\hline
\end{DoxyTemplParams}
This class represents a pseudo expression with broadcasting and partial reduction features. It is the return type of Dense\+Base\+::colwise() and Dense\+Base\+::rowwise() and most of the time this is the only way it is explicitly used.

To understand the logic of rowwise/colwise expression, let\textquotesingle{}s consider a generic case {\ttfamily A.\+colwise().foo()} where {\ttfamily foo} is any method of {\ttfamily \mbox{\hyperlink{class_eigen_1_1_vectorwise_op}{Vectorwise\+Op}}}. This expression is equivalent to applying {\ttfamily foo()} to each column of {\ttfamily A} and then re-\/assemble the outputs in a matrix expression\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{[A.col(0).foo(), A.col(1).foo(), ..., A.col(A.cols()-\/1).foo()] }
\end{DoxyCode}

For a partial reduction on an empty input, some rules apply. For the sake of clarity, let\textquotesingle{}s consider a vertical reduction\+:
\begin{DoxyItemize}
\item If the number of columns is zero, then a 1x0 row-\/major vector expression is returned.
\item Otherwise, if the number of rows is zero, then
\begin{DoxyItemize}
\item a row vector of zeros is returned for sum-\/like reductions (sum, squared\+Norm, norm, etc.)
\item a row vector of ones is returned for a product reduction (e.\+g., {\ttfamily Matrix\+Xd(n,0).colwise().\mbox{\hyperlink{class_eigen_1_1_vectorwise_op_a8dcc44dca1f1f973cfd5373183c50f91}{prod()}}})
