If the Path\+Effect can be represented as a dash pattern, as\+A\+Dash will return k\+Dash\+\_\+\+Dash\+Type and None otherwise. If a non N\+U\+LL info is passed in, the various Dash\+Info will be filled in if the Path\+Effect can be a dash pattern. If passed in info has an f\+Count equal or greater to that of the effect, it will memcpy the values of the dash intervals into the info. Thus the general approach will be call as\+A\+Dash once with default info to get Dash\+Type and f\+Count. If effect can be represented as a dash pattern, allocate space for the intervals in info, then call as\+A\+Dash again with the same info and the intervals will get copied in.
