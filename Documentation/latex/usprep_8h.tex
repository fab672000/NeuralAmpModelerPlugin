String\+Prep A\+PI implements the Sting\+Prep framework as described by R\+FC 3454. String\+Prep prepares Unicode strings for use in network protocols. Profiles of Sting\+Prep are set of rules and data according to with the Unicode Strings are prepared. Each profiles contains tables which describe how a code point should be treated. The tables are broadly classified into 
\begin{DoxyItemize}
\item Unassigned Table\+: Contains code points that are unassigned in the Unicode \mbox{\hyperlink{struct_version}{Version}} supported by String\+Prep. Currently R\+FC 3454 supports Unicode 3.\+2.  
\item Prohibited Table\+: Contains code points that are prohibited from the output of the String\+Prep processing function.  
\item Mapping Table\+: Contains code points that are deleted from the output or case mapped.  
\end{DoxyItemize}

The procedure for preparing Unicode strings\+: 
\begin{DoxyEnumerate}
\item Map\+: For each character in the input, check if it has a mapping and, if so, replace it with its mapping.  
\item Normalize\+: Possibly normalize the result of step 1 using Unicode normalization.  
\item Prohibit\+: Check for any characters that are not allowed in the output. If any are found, return an error. 
\item Check bidi\+: Possibly check for right-\/to-\/left characters, and if any are found, make sure that the whole string satisfies the requirements for bidirectional strings. If the string does not satisfy the requirements for bidirectional strings, return an error.
