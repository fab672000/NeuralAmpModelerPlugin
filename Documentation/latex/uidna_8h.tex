\doxysubsection{Detailed Description}
C A\+PI\+: Internationalizing Domain Names in Applications (I\+D\+NA) 

I\+D\+N\+A2008 is implemented according to U\+TS \#46, see the I\+D\+NA C++ class in \mbox{\hyperlink{idna_8h}{idna.\+h}}.

I\+D\+N\+A2003\+: This function implements the To\+A\+S\+C\+II operation as defined in the I\+D\+NA R\+FC. This operation is done on {\bfseries{single labels}} before sending it to something that expects A\+S\+C\+II names. A label is an individual part of a domain name. Labels are usually separated by dots; e.\+g. \char`\"{}www.\+example.\+com\char`\"{} is composed of 3 labels \char`\"{}www\char`\"{},\char`\"{}example\char`\"{}, and \char`\"{}com\char`\"{}.

I\+D\+N\+A2003 A\+PI Overview\+:

The uidna\+\_\+ A\+PI implements the I\+D\+NA protocol as defined in the I\+D\+NA R\+FC (\href{http://www.ietf.org/rfc/rfc3490.txt}{\texttt{ http\+://www.\+ietf.\+org/rfc/rfc3490.\+txt}}). The R\+FC defines 2 operations\+: To\+A\+S\+C\+II and To\+Unicode. Domain name labels containing non-\/\+A\+S\+C\+II code points are processed by the To\+A\+S\+C\+II operation before passing it to resolver libraries. Domain names that are obtained from resolver libraries are processed by the To\+Unicode operation before displaying the domain name to the user. I\+D\+NA requires that implementations process input strings with Nameprep (\href{http://www.ietf.org/rfc/rfc3491.txt}{\texttt{ http\+://www.\+ietf.\+org/rfc/rfc3491.\+txt}}), which is a profile of Stringprep (\href{http://www.ietf.org/rfc/rfc3454.txt}{\texttt{ http\+://www.\+ietf.\+org/rfc/rfc3454.\+txt}}), and then with Punycode (\href{http://www.ietf.org/rfc/rfc3492.txt}{\texttt{ http\+://www.\+ietf.\+org/rfc/rfc3492.\+txt}}). Implementations of I\+D\+NA M\+U\+ST fully implement Nameprep and Punycode; neither Nameprep nor Punycode are optional. The input and output of To\+A\+S\+C\+II and To\+Unicode operations are Unicode and are designed to be chainable, i.\+e., applying To\+A\+S\+C\+II or To\+Unicode operations multiple times to an input string will yield the same result as applying the operation once. To\+Unicode(To\+Unicode(To\+Unicode...(To\+Unicode(string)))) == To\+Unicode(string) To\+A\+S\+C\+II(To\+A\+S\+C\+II(To\+A\+S\+C\+II...(To\+A\+S\+C\+I\+I(string))) == To\+A\+S\+C\+I\+I(string).
