Oboe streams on Android 9 (Pie) and above can use the Java/\+Kotlin. See \href{https://developer.android.com/reference/android/media/audiofx/AudioEffect}{\texttt{ Audio\+Effect A\+PI}}

The basic idea is to use Java or Kotlin to create a Session with Effects. Then associate your Oboe streams with the session by creating them with a Session\+ID.

In Java\+: \begin{DoxyVerb}AudioManager audioManager = (AudioManager) context.getSystemService(Context.AUDIO_SERVICE);
int audioSessionId = audioManager.generateAudioSessionId();
\end{DoxyVerb}


Pass the audio\+Session\+Id to your C++ code using J\+NI. Then use it when opening your Oboe streams\+: \begin{DoxyVerb}builder->setSessionId(sessionId);
\end{DoxyVerb}


Note that these streams will probably not have low latency. So you may want to do your own effects processing.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_00ac400e25ebf1996ffa981ed0371880_autotoc_md433}{}\doxysubsection{Using Third Party Affects Processing}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_00ac400e25ebf1996ffa981ed0371880_autotoc_md433}
