The Chorus effect makes copies of its input sound and remixes them to the stereo field using differently modulated delays, thus simulating a chorus of nearly identical sound sources.

The Chorus output is always stereo, while its input may be either mono or stereo. To minimize C\+PU usage, place the Chorus at the point in your effects chain where the effects are mono to the left and stereo to the right.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_df114fe5935eda2a01d303fbc71a60ae_autotoc_md901}{}\doxysection{Parameters}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_df114fe5935eda2a01d303fbc71a60ae_autotoc_md901}

The faust code (.dsp) in this directory has been compiled using faust2sam into chorus-\/sam


\begin{DoxyCode}{0}
\DoxyCodeLine{\% faust2sam -\/midi chorus.dsp}
\end{DoxyCode}


The result of this compile is a zip file {\ttfamily chorus-\/sam.\+zip} containing the following three files\+:


\begin{DoxyItemize}
\item {\ttfamily fast\+\_\+pow2.\+h}
\item {\ttfamily sam\+Faust\+D\+S\+P.\+cpp}
\item {\ttfamily sam\+Faust\+D\+S\+P.\+h}
\end{DoxyItemize}

These 3 files should be placed into the directory {\ttfamily sam\+\_\+baremetal\+\_\+framework\+\_\+core1/src/faust}

In addition there is a header file that is common across all cores called {\ttfamily audio\+\_\+system\+\_\+config.\+h}. In this file the following pre-\/processor variables should be set in the following way. The example below indicates that a \mbox{\hyperlink{struct_faust}{Faust}} algorithm will only be running on Core1 and that Core2 will be simply passing audio to the codec.

\DoxyCodeLine{\#define USE\_FAUST\_ALGORITHM\_CORE2           FALSE}
