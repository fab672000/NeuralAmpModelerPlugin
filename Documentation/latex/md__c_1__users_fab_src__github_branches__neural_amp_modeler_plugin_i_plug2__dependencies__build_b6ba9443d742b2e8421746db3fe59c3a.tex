The O\+SC address space adheres strictly to the hierarchy defined by the \textquotesingle{}addnode\textquotesingle{} and \textquotesingle{}opengroup\textquotesingle{} calls. A node expects to receive O\+SC messages with a single float value as parameter. This policy is strict for the parameters count, but relaxed for the parameter type\+: O\+SC int values are accepted and casted to float.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md949}{}\doxysubsubsection{\textquotesingle{}get\textquotesingle{} message}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md949}
The library provides a simple support for querying the system with a \textquotesingle{}get\textquotesingle{} message that can be sent to any valid O\+SC address. The \textquotesingle{}get\textquotesingle{} message is propagated to every terminal node that responds with its osc address and current values (value, min and max).\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md950}{}\doxysubsubsection{\textquotesingle{}hello\textquotesingle{} message}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md950}
The library provides a simple support for modules discovery with a \textquotesingle{}hello\textquotesingle{} message that can be sent to any module root address. The module responds with its root address, followed by its IP address, followed by the U\+DP ports numbers (listening port, output port, error port). See the note about network management below for ports numbering scheme.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md951}{}\doxysubsubsection{\textquotesingle{}json\textquotesingle{} message}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md951}
The {\ttfamily json} message is handled by any module root address. The J\+S\+ON module description will be sent on the output port.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md952}{}\doxysubsubsection{\textquotesingle{}xmit\textquotesingle{} message}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md952}
The {\ttfamily xmit} message is handled by any module root address. It must be followed by an integer value (Ø$\vert$1$\vert$2). It turns the O\+SC \mbox{\hyperlink{class_u_i}{UI}} transmit mode on or all or alias. When {\ttfamily all}, any change of a \mbox{\hyperlink{class_u_i}{UI}} element generates the corresponding O\+SC message including aliases messages if any and the module could be used as a remote control for similar modules. In {\ttfamily alias} mode, only the aliases messages are sent.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md953}{}\doxysubsubsection{\textquotesingle{}bundle\textquotesingle{} message}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md953}
The {\ttfamily bundle} message is handled by any module root address. It must be followed by a boolean value (Ø$\vert$1). The {\ttfamily bundle} mode affects the transmition mode (see {\ttfamily xmit} message above). When the {\ttfamily bundle} mode is on, all the O\+SC messages are generated in an O\+SC bundle that must be sent at regular interval. To do so, the \mbox{\hyperlink{class_o_s_c_u_i}{O\+S\+C\+UI}} class provides an {\ttfamily end\+Bundle()} method, it\textquotesingle{}s the architecture responsability to call this method to trigger the current bundle transmission. A typical implementation could call it just after the compute method.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md954}{}\doxysubsubsection{\textquotesingle{}alias\textquotesingle{} message}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md954}
The {\ttfamily alias} mechanism provided at metadata level to control a \mbox{\hyperlink{struct_faust}{Faust}} application from a preexisting set of O\+SC messages, is dynamically available using O\+SC message as well. The general form of the O\+SC {\ttfamily alias} message is the following\+:

\DoxyCodeLine{alias message :=}
\DoxyCodeLine{  a)                /target/address alias}
\DoxyCodeLine{  b)              | /target/address alias /alias/address}
\DoxyCodeLine{  c)              | /target/address alias /alias/address min max}

\begin{DoxyItemize}
\item The {\bfseries{a)}} form removes all aliases associated to {\ttfamily /target/address}
\item The {\bfseries{b)}} form removes the aliase {\ttfamily /alias/address} associated to {\ttfamily /target/address}
\item The {\bfseries{c)}} form creates the alias {\ttfamily /alias/address} to {\ttfamily /target/address} and scales the alias values from {\ttfamily min}, {\ttfamily max} to the target range of values.

The default U\+DP ports are chosen in an unassigned range see I\+A\+NA P\+O\+RT N\+U\+M\+B\+E\+RS (last updated 2011-\/01-\/24) at \href{http://www.iana.org/assignments/port-numbers}{\texttt{ http\+://www.\+iana.\+org/assignments/port-\/numbers}} As of today (jan. 27 2011), the range 5507-\/5552 is unassigned and the default ports numbers are\+:
\begin{DoxyItemize}
\item {\bfseries{5510}} the listening U\+DP port number
\item {\bfseries{5511}} the O\+SC output port number
\item {\bfseries{5512}} the O\+SC error port number

An O\+S\+C\+Controler object takes the command line arguments as constructor parameters. It supports the following options to change the U\+DP ports numbers\+: \mbox{[}-\/port $\vert$ -\/outport $\vert$ -\/errport\mbox{]} number\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md957}{}\doxysubsubsection{Dynamic U\+D\+P listening port allocation}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md957}
When the U\+DP listening port number is busy, the system automatically looks for the next available port number. Unless otherwise specified by the command line, the U\+DP output port numbers are unchanged. An O\+SC application sends the allocated ports numbers on the O\+SC output port on startup.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md958}{}\doxysubsubsection{Destination address}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md958}
The default U\+DP output streams destination is \textquotesingle{}localhost\textquotesingle{}. It can be changed with a command line option\+: -\/dest address where address is a host name os an IP number.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md959}{}\doxysection{Sharing ports numbers between several applications}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_b6ba9443d742b2e8421746db3fe59c3a_autotoc_md959}
Since version 1.\+1.\+0 of the \mbox{\hyperlink{struct_faust}{Faust}} O\+SC library, a new command line option \mbox{[} -\/reuse \mbox{]} allows to share a listening port between several applications on the same host. It works in conjunction with the multicast address 224.\+0.\+0.\+1. Thus, messages addressed to applications launched with {\ttfamily -\/reuse 1} must be send to the IP address 224.\+0.\+0.\+1.

{\bfseries{Note}}\+: any application running on any host on the same local network, launched with {\ttfamily -\/reuse 1} and using the same listening port, will receive all the messages addressed to 224.\+0.\+0.\+1 on this port.

Different listening port numbers can be used to create a group of applications restricted to a local host or to create an arbitrary group of applications distributed over the local network.
