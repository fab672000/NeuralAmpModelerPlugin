\doxysubsection{Detailed Description}
\mbox{\hyperlink{struct_u_data_info}{U\+Data\+Info}} contains the properties about the requested data. This is meta data.

This structure may grow in the future, indicated by the {\ttfamily size} field.

I\+CU data must be at least 8-\/aligned, and should be 16-\/aligned. The \mbox{\hyperlink{struct_u_data_info}{U\+Data\+Info}} struct begins 4 bytes after the start of the data item, so it is 4-\/aligned.

The platform data property fields help determine if a data file can be efficiently used on a given machine. The particular fields are of importance only if the data is affected by the properties -\/ if there is integer data with word sizes $>$ 1 byte, char$\ast$ text, or U\+Char$\ast$ text.

The implementation for the {\ttfamily udata\+\_\+open\mbox{[}Choice\mbox{]}()} functions may reject data based on the value in {\ttfamily is\+Big\+Endian}. No other field is used by the {\ttfamily udata} A\+PI implementation.

The {\ttfamily data\+Format} may be used to identify the kind of data, e.\+g. a converter table.

The {\ttfamily format\+Version} field should be used to make sure that the format can be interpreted. It may be a good idea to check only for the one or two highest of the version elements to allow the data memory to get more or somewhat rearranged contents, for as long as the using code can still interpret the older contents.

The {\ttfamily data\+Version} field is intended to be a common place to store the source version of the data; for data from the Unicode character database, this could reflect the Unicode version.
