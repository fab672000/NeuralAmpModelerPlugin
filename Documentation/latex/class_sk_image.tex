\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_sk_image}{Sk\+Image}} describes a two dimensional array of pixels to draw. The pixels may be decoded in a raster bitmap, encoded in a \mbox{\hyperlink{class_sk_picture}{Sk\+Picture}} or compressed data stream, or located in G\+PU memory as a G\+PU texture.

\mbox{\hyperlink{class_sk_image}{Sk\+Image}} cannot be modified after it is created. \mbox{\hyperlink{class_sk_image}{Sk\+Image}} may allocate additional storage as needed; for instance, an encoded \mbox{\hyperlink{class_sk_image}{Sk\+Image}} may decode when drawn.

\mbox{\hyperlink{class_sk_image}{Sk\+Image}} width and height are greater than zero. Creating an \mbox{\hyperlink{class_sk_image}{Sk\+Image}} with zero width or height returns \mbox{\hyperlink{class_sk_image}{Sk\+Image}} equal to nullptr.

\mbox{\hyperlink{class_sk_image}{Sk\+Image}} may be created from \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}}, \mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}}, \mbox{\hyperlink{class_sk_surface}{Sk\+Surface}}, \mbox{\hyperlink{class_sk_picture}{Sk\+Picture}}, encoded streams, G\+PU texture, Y\+U\+V\+\_\+\+Color\+Space data, or hardware buffer. Encoded streams supported include B\+MP, G\+IF, H\+E\+IF, I\+CO, J\+P\+EG, P\+NG, W\+B\+MP, WebP. Supported encoding details vary with platform.

Makes image pixel data available to caller, possibly asynchronously. It can also rescale the image pixels.

Currently asynchronous reads are only supported on the G\+PU backend and only when the underlying 3D A\+PI supports transfer buffers and C\+P\+U/\+G\+PU synchronization primitives. In all other cases this operates synchronously.

Data is read from the source sub-\/rectangle, is optionally converted to a linear gamma, is rescaled to the size indicated by \textquotesingle{}info\textquotesingle{}, is then converted to the color space, color type, and alpha type of \textquotesingle{}info\textquotesingle{}. A \textquotesingle{}src\+Rect\textquotesingle{} that is not contained by the bounds of the image causes failure.

When the pixel data is ready the caller\textquotesingle{}s Read\+Pixels\+Callback is called with a \mbox{\hyperlink{class_sk_image_1_1_async_read_result}{Async\+Read\+Result}} containing pixel data in the requested color type, alpha type, and color space. The \mbox{\hyperlink{class_sk_image_1_1_async_read_result}{Async\+Read\+Result}} will have count() == 1. Upon failure the callback is called with nullptr for \mbox{\hyperlink{class_sk_image_1_1_async_read_result}{Async\+Read\+Result}}. For a G\+PU image this flushes work but a submit must occur to guarantee a finite time before the callback is called.

The data is valid for the lifetime of \mbox{\hyperlink{class_sk_image_1_1_async_read_result}{Async\+Read\+Result}} with the exception that if the \mbox{\hyperlink{class_sk_image}{Sk\+Image}} is G\+P\+U-\/backed the data is immediately invalidated if the context is abandoned or destroyed.
