\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_sk_field_visitor}{Sk\+Field\+Visitor}} is an interface that can be implemented by any class to visit all fields of \mbox{\hyperlink{class_sk_reflected}{Sk\+Reflected}} types, and of types that implement the visit\+Fields() function.

Classes implementing the interface must supply implementations of virtual functions that visit basic types (float, int, bool, \mbox{\hyperlink{class_sk_string}{Sk\+String}}), as well as helper methods for entering the scope of an object or array.

All visit functions supply a field name, and a non-\/constant reference to an actual field. This allows visitors to serialize or deserialize collections of objects, or perform edits on existing objects.

Classes that implement visit\+Fields (typically derived from \mbox{\hyperlink{class_sk_reflected}{Sk\+Reflected}}) should simply call visit() for each of their fields, passing a (unique) field name, and the actual field. If your class has derived fields, it\textquotesingle{}s best to only visit() the fields that you would serialize, then enforce any constraints afterwards.

See \mbox{\hyperlink{_sk_particle_serialization_8h_source}{Sk\+Particle\+Serialization.\+h}} for example visitors that perform serialization to and from J\+S\+ON.
