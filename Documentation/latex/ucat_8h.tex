\doxysubsection{Detailed Description}
C A\+PI\+: Message Catalog Wrappers. 

This C A\+PI provides look-\/alike functions that deliberately resemble the P\+O\+S\+IX catopen, catclose, and catgets functions. The underlying implementation is in terms of I\+CU resource bundles, rather than P\+O\+S\+IX message catalogs.

The I\+CU resource bundles obey standard I\+CU inheritance policies. To facilitate this, sets and messages are flattened into one tier. This is done by creating resource bundle keys of the form $<$set\+\_\+num$>$\%$<$msg\+\_\+num$>$ where set\+\_\+num is the set number and msg\+\_\+num is the message number, formatted as decimal strings.

Example\+: Consider a message catalog containing two sets\+:

Set 1\+: Message 4 = \char`\"{}\+Good morning.\char`\"{} Message 5 = \char`\"{}\+Good afternoon.\char`\"{} Message 7 = \char`\"{}\+Good evening.\char`\"{} Message 8 = \char`\"{}\+Good night.\char`\"{} Set 4\+: Message 14 = \char`\"{}\+Please \char`\"{} Message 19 = \char`\"{}\+Thank you.\char`\"{} Message 20 = \char`\"{}\+Sincerely,\char`\"{}

The I\+CU resource bundle source file would, assuming it is named \char`\"{}greet.\+txt\char`\"{}, would look like this\+:

greet \{ 1\%4 \{ \char`\"{}\+Good morning.\char`\"{} \} 1\%5 \{ \char`\"{}\+Good afternoon.\char`\"{} \} 1\%7 \{ \char`\"{}\+Good evening.\char`\"{} \} 1\%8 \{ \char`\"{}\+Good night.\char`\"{} \}

4\%14 \{ \char`\"{}\+Please \char`\"{} \} 4\%19 \{ \char`\"{}\+Thank you.\char`\"{} \} 4\%20 \{ \char`\"{}\+Sincerely,\char`\"{} \} \}

The catgets function is commonly used in combination with functions like printf and strftime. I\+CU components like message format can be used instead, although they use a different format syntax. There is an I\+CU package, icuio, that provides some of the P\+O\+S\+I\+X-\/style formatting A\+PI.
