\doxysubsection{Detailed Description}
Implements \mbox{\hyperlink{class_steinberg_1_1_f_unknown}{F\+Unknown}} and \mbox{\hyperlink{class_steinberg_1_1_i_dependent}{I\+Dependent}}.

\mbox{\hyperlink{class_steinberg_1_1_f_object}{F\+Object}} is a polymorphic class that implements \mbox{\hyperlink{class_steinberg_1_1_i_dependent}{I\+Dependent}} (of S\+KI module) and therefore derived from \mbox{\hyperlink{class_steinberg_1_1_f_unknown}{F\+Unknown}}, which is the most abstract base class of all.

All C\+O\+M-\/like virtual methods of \mbox{\hyperlink{class_steinberg_1_1_f_unknown}{F\+Unknown}} such as \mbox{\hyperlink{class_steinberg_1_1_f_object_aa84d396d241ebe17f94bacec33f9491b}{query\+Interface()}}, \mbox{\hyperlink{class_steinberg_1_1_f_object_a4527f132b11157d2531a355165f8fc00}{add\+Ref()}}, \mbox{\hyperlink{class_steinberg_1_1_f_object_ac7145cd684568b59bc534750cb556f4f}{release()}} are implemented here. On top of that, dependency-\/related methods are implemented too.

Pointer casting is done via the template methods F\+Cast, either \mbox{\hyperlink{class_steinberg_1_1_f_object}{F\+Object}} to \mbox{\hyperlink{class_steinberg_1_1_f_object}{F\+Object}} or \mbox{\hyperlink{class_steinberg_1_1_f_unknown}{F\+Unknown}} to \mbox{\hyperlink{class_steinberg_1_1_f_object}{F\+Object}}.

\mbox{\hyperlink{class_steinberg_1_1_f_object}{F\+Object}} supports a new singleton concept, therefore these objects are deleted automatically upon program termination.
