\doxysubsection{Detailed Description}
Provides the abstract interface for enumerating definitions. 

\begin{DoxyRemark}{Remarks}

\end{DoxyRemark}
\begin{DoxyNote}{Note}
This \char`\"{}\+Enumeration\char`\"{} interface is different from \char`\"{}standard\char`\"{} C\+OM for several reasons\+:
\begin{DoxyEnumerate}
\item Enumerations of related interfaces are expected to support the same type of definitions interface.
\item There no \char`\"{}partial success\char`\"{} in A\+CF. If the caller asks for the Next 10 definitions and there are only 9 Next method will F\+A\+IL. If all of the requested definitions do not support the given iid then the Next method will F\+A\+IL. If the caller asks for 0 definitions then the Next method will F\+A\+IL. (R\+E\+S\+U\+LT codes\+:T\+BD)
\item Unlike the \char`\"{}standard\char`\"{} C\+OM enumeration interface this interface could actually be remoted without an a special \char`\"{}\+Remote\+As\char`\"{} method in a proxy dll. (Of course we have no plans to ever remote A\+CF interfaces...) 
\end{DoxyEnumerate}

This interface is implemented by the host. A plug-\/in will use this interface to access of related definitions. ~\newline
