\begin{DoxyNote}{Note}
This truncation is N\+OT identical to C style casting. Because the Intel (and \mbox{\hyperlink{struct_i}{I}} would assume Power\+PC) processors use convergent rounding by default, exactly whole odd numbers will truncate down by 1.\+0 (e.\+g. 0.\+0-\/$>$0, 1.\+0-\/$>$0, 2.\+0-\/$>$2, 3.\+0-\/$>$2). Surprisingly, even with these limitations this fast float to int conversion is often very useful in practice, as long as one is aware of these issues.
