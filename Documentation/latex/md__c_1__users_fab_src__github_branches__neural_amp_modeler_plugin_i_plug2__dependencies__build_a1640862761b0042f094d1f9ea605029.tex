Oboe is a C++ library which makes it easy to build high-\/performance audio apps on Android. It was created primarily to allow developers to target a simplified A\+PI that works across multiple A\+PI levels back to A\+PI level 16 (Jelly Bean).\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_a1640862761b0042f094d1f9ea605029_autotoc_md442}{}\doxysection{Features}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_a1640862761b0042f094d1f9ea605029_autotoc_md442}

\begin{DoxyItemize}
\item Compatible with A\+PI 16 onwards -\/ runs on 99\% of Android devices
\item Chooses the audio A\+PI (Open\+SL ES on A\+PI 16+ or A\+Audio on A\+PI 27+) which will give the best audio performance on the target Android device
\item Automatic latency tuning
\item Modern C++ allowing you to write clean, elegant code
\item Used by popular apps and frameworks
\end{DoxyItemize}\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_a1640862761b0042f094d1f9ea605029_autotoc_md443}{}\doxysection{Requirements}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_a1640862761b0042f094d1f9ea605029_autotoc_md443}
To build Oboe you\textquotesingle{}ll need a compiler which supports C++14 and the Android header files. The easiest way to obtain these is by downloading the Android N\+DK r17 or above. It can be installed using Android Studio\textquotesingle{}s S\+DK manager, or via \href{https://developer.android.com/ndk/downloads/}{\texttt{ direct download}}.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_a1640862761b0042f094d1f9ea605029_autotoc_md444}{}\doxysection{A\+P\+I Documentation}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_a1640862761b0042f094d1f9ea605029_autotoc_md444}
