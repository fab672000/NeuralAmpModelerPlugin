\doxysubsection{Detailed Description}
This header provides classes for working with \mbox{\hyperlink{class_steinberg_1_1_f_unknown}{F\+Unknown}}.

An interface which shall support Steinbergs Module Architecture should inherit from U\+::\+Unknown and provide a typedef {\ttfamily I\+ID} of type U\+::\+U\+ID.

On OS X you can generate an U\+::\+U\+ID and copy it to the clipboard with the following shell command\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{uuidgen | \{ read id; echo -\/n \textcolor{stringliteral}{"using IID = U::UID<0x\$\{id:0:8\}, 0x\$\{id:9:4\}\$\{id:14:4\}, 0x\$\{id:19:4\}\$\{id:24:4\}, 0x\$\{id:28:8\}>;"} ; \} | pbcopy}
\end{DoxyCode}


Example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{struct }IFoo : \textcolor{keyword}{public} U::Unknown}
\DoxyCodeLine{\{}
\DoxyCodeLine{    \textcolor{comment}{// Use a generated random uid here.}}
\DoxyCodeLine{    \textcolor{keyword}{using} IID = U::UID<0x01234567, 0x89012345, 0x67890123, 0x45678901>;}
\DoxyCodeLine{}
\DoxyCodeLine{    \textcolor{keyword}{virtual} \textcolor{keywordtype}{void} bar () = 0;}
\DoxyCodeLine{\};}
\end{DoxyCode}


A class implementing the interface {\ttfamily I\+Foo} uses the U\+::\+Implements template to specify the interfaces it implements. All interfaces which the class should derive from need to be listed in the U\+::\+Directly template.

Example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{struct }FooImpl : \textcolor{keyword}{public} U::Implements<U::Directly<IFoo>>}
\DoxyCodeLine{\{}
\DoxyCodeLine{    \textcolor{keywordtype}{void} bar ()\textcolor{keyword}{ override }\{\}}
\DoxyCodeLine{\};}
\end{DoxyCode}


To check if a class can provide a specific interface use the U\+::cast function.

Example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{void} test (U::Unknown* obj)}
\DoxyCodeLine{\{}
\DoxyCodeLine{    \textcolor{keywordflow}{if} (\textcolor{keyword}{auto} foo = U::cast<IFoo> (obj))}
\DoxyCodeLine{    \{}
\DoxyCodeLine{        \textcolor{comment}{// obj provided IFoo}}
\DoxyCodeLine{    \}}
\DoxyCodeLine{\}}
\end{DoxyCode}


The U\+::\+Implements class supports a second template parameter U\+::\+Indirectly for specifying a list of interfaces which should be available via {\ttfamily query\+Interface} but not inherited from. This is useful if an interface extends another interface.

Example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{struct }IBar : \textcolor{keyword}{public} IFoo}
\DoxyCodeLine{\{}
\DoxyCodeLine{    \textcolor{keyword}{using} IID = U::UID<0x11223344, 0x55667788, 0x99001122, 0x33445566>;}
\DoxyCodeLine{}
\DoxyCodeLine{    \textcolor{keyword}{virtual} \textcolor{keywordtype}{void} baz () = 0;}
\DoxyCodeLine{\};}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keyword}{struct }BarImpl : \textcolor{keyword}{public} U::Implements<U::Directly<IBar>, U::Indirectly<IFoo>>}
\DoxyCodeLine{\{}
\DoxyCodeLine{    \textcolor{keywordtype}{void} bar ()\textcolor{keyword}{ override }\{\}}
\DoxyCodeLine{    \textcolor{keywordtype}{void} baz ()\textcolor{keyword}{ override }\{\}}
\DoxyCodeLine{\};}
\end{DoxyCode}


In some cases a class shall be extended and an additional interface implemented. This is possible with the U\+::\+Extends template which is a generalization of the U\+::\+Implements template and allows specifying a base class from which should be inherited.

Example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{struct }ITest : \textcolor{keyword}{public} U::Unknown}
\DoxyCodeLine{\{}
\DoxyCodeLine{    \textcolor{keyword}{using} IID = U::UID<0x99887766, 0x55443322, 0x11009988, 0x77665544>;}
\DoxyCodeLine{}
\DoxyCodeLine{    \textcolor{keyword}{virtual} \textcolor{keywordtype}{bool} \mbox{\hyperlink{group__core__func__vector__relational_ga229aa6c48f93155ae6547210e005f06c}{equal}} (\textcolor{keywordtype}{int} a, \textcolor{keywordtype}{int} b) \textcolor{keyword}{const} = 0;}
\DoxyCodeLine{\};}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keyword}{struct }TestImpl : \textcolor{keyword}{public} U::Extends<FooImpl, U::Directly<ITest>>}
\DoxyCodeLine{\{}
\DoxyCodeLine{    \textcolor{keywordtype}{bool} \mbox{\hyperlink{group__core__func__vector__relational_ga229aa6c48f93155ae6547210e005f06c}{equal}} (\textcolor{keywordtype}{int} a, \textcolor{keywordtype}{int} b)\textcolor{keyword}{ const override }\{ \textcolor{keywordflow}{return} a == b; \}}
\DoxyCodeLine{\};}
\end{DoxyCode}


To pass arbitrary arguments to the specified base class one can use the inherited {\ttfamily Base} typedef. All arguments passed to {\ttfamily Base} are automatically perfectly forwarded to the base class.

In the following example the value 42 is passed to the {\ttfamily Alternative\+Foo\+Impl} base class\+:

\DoxyCodeLine{}
\DoxyCodeLine{    \textcolor{keywordtype}{int} dummy;}
\DoxyCodeLine{\};}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keyword}{struct }AlternativeTestImpl : \textcolor{keyword}{public} U::Extends<AlternativeFooImpl, U::Directly<ITest>>}
\DoxyCodeLine{\{}
\DoxyCodeLine{    AlternativeTestImpl () : Base \{42\} \{\}}

A replacement for Init\+Module and Deinit\+Module

If you link this file the Init\+Module and Deinit\+Module functions are implemented and you can use this to register functions that will be called when the module is loaded and before the module is unloaded.

Use this for one time initializers or cleanup functions. For example\+: if you depend on a 3rd party library that needs initialization before you can use it you can write an initializer like this\+:

static \mbox{\hyperlink{struct_steinberg_1_1_module_initializer}{Module\+Initializer}} Init\+My\+External\+Lib (\mbox{[}\mbox{]} () \{ My\+External\+Lib\+::init (); \});

Or you have a lazy create wavetable you need to free the allocated memory later\+:

static \mbox{\hyperlink{struct_steinberg_1_1_module_terminator}{Module\+Terminator}} Free\+Wave\+Table\+Memory (\mbox{[}\mbox{]} () \{ My\+Wave\+Table\+::free (); \});
