\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Scalar\+\_\+, class \+\_\+\+System$>$\newline
class Eigen\+::\+Euler\+Angles$<$ Scalar\+\_\+, \+\_\+\+System $>$}

Represents a rotation in a 3 dimensional space as three Euler angles. 

Euler rotation is a set of three rotation of three angles over three fixed axes, defined by the \mbox{\hyperlink{class_eigen_1_1_euler_system}{Euler\+System}} given as a template parameter.

Here is how intrinsic Euler angles works\+:
\begin{DoxyItemize}
\item first, rotate the axes system over the alpha axis in angle alpha
\item then, rotate the axes system over the beta axis(which was rotated in the first stage) in angle beta
\item then, rotate the axes system over the gamma axis(which was rotated in the two stages above) in angle gamma
\end{DoxyItemize}

\begin{DoxyNote}{Note}
This class support only intrinsic Euler angles for simplicity, see \mbox{\hyperlink{class_eigen_1_1_euler_system}{Euler\+System}} how to easily overcome this for extrinsic systems.
\end{DoxyNote}
\hypertarget{class_eigen_1_1_euler_angles_autotoc_md1552}{}\doxyparagraph{Rotation representation and conversions}\label{class_eigen_1_1_euler_angles_autotoc_md1552}
It has been proved(see Wikipedia link below) that every rotation can be represented by Euler angles, but there is no single representation (e.\+g. unlike rotation matrices). Therefore, you can convert from \mbox{\hyperlink{namespace_eigen}{Eigen}} rotation and to them (including rotation matrices, which is not called \char`\"{}rotations\char`\"{} by \mbox{\hyperlink{namespace_eigen}{Eigen}} design).

Euler angles usually used for\+:
\begin{DoxyItemize}
\item convenient human representation of rotation, especially in interactive \mbox{\hyperlink{class_g_u_i}{G\+UI}}.
\item gimbal systems and robotics
\item efficient encoding(i.\+e. 3 floats only) of rotation for network protocols.
\end{DoxyItemize}

However, Euler angles are slow comparing to quaternion or matrices, because their unnatural math definition, although it\textquotesingle{}s simple for human. To overcome this, this class provide easy movement from the math friendly representation to the human friendly representation, and vise-\/versa.

All the user need to do is a safe simple C++ type conversion, and this class take care for the math. Additionally, some axes related computation is done in compile time.\hypertarget{class_eigen_1_1_euler_angles_autotoc_md1553}{}\doxysubparagraph{Euler angles ranges in conversions}\label{class_eigen_1_1_euler_angles_autotoc_md1553}
Rotations representation as \mbox{\hyperlink{class_eigen_1_1_euler_angles}{Euler\+Angles}} are not single (unlike matrices), and even have infinite \mbox{\hyperlink{class_eigen_1_1_euler_angles}{Euler\+Angles}} representations.~\newline
 For example, add or subtract 2$\ast$\+PI from either angle of \mbox{\hyperlink{class_eigen_1_1_euler_angles}{Euler\+Angles}} and you\textquotesingle{}ll get the same rotation. This is the general reason for infinite representation, but it\textquotesingle{}s not the only general reason for not having a single representation.

When converting rotation to \mbox{\hyperlink{class_eigen_1_1_euler_angles}{Euler\+Angles}}, this class convert it to specific ranges When converting some rotation to \mbox{\hyperlink{class_eigen_1_1_euler_angles}{Euler\+Angles}}, the rules for ranges are as follow\+:
\begin{DoxyItemize}
\item If the rotation we converting from is an \mbox{\hyperlink{class_eigen_1_1_euler_angles}{Euler\+Angles}} (even when it represented as \mbox{\hyperlink{class_eigen_1_1_rotation_base}{Rotation\+Base}} explicitly), angles ranges are {\bfseries{undefined}}.
\item otherwise, alpha and gamma angles will be in the range \mbox{[}-\/PI, PI\mbox{]}.~\newline
 As for Beta angle\+:
\begin{DoxyItemize}
\item If the system is Tait-\/\+Bryan, the beta angle will be in the range \mbox{[}-\/P\+I/2, P\+I/2\mbox{]}.
\item otherwise\+:
\begin{DoxyItemize}
\item If the beta axis is positive, the beta angle will be in the range \mbox{[}0, PI\mbox{]}
\item If the beta axis is negative, the beta angle will be in the range \mbox{[}-\/PI, 0\mbox{]}
\end{DoxyItemize}
\end{DoxyItemize}
\end{DoxyItemize}

\begin{DoxyItemize}

\end{DoxyItemize}
