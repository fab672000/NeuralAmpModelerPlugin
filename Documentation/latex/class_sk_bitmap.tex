\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} describes a two-\/dimensional raster pixel array. \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} is built on \mbox{\hyperlink{struct_sk_image_info}{Sk\+Image\+Info}}, containing integer width and height, Sk\+Color\+Type and Sk\+Alpha\+Type describing the pixel format, and \mbox{\hyperlink{class_sk_color_space}{Sk\+Color\+Space}} describing the range of colors. \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} points to \mbox{\hyperlink{class_sk_pixel_ref}{Sk\+Pixel\+Ref}}, which describes the physical array of pixels. \mbox{\hyperlink{struct_sk_image_info}{Sk\+Image\+Info}} bounds may be located anywhere fully inside \mbox{\hyperlink{class_sk_pixel_ref}{Sk\+Pixel\+Ref}} bounds.

\mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} can be drawn using \mbox{\hyperlink{class_sk_canvas}{Sk\+Canvas}}. \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} can be a drawing destination for \mbox{\hyperlink{class_sk_canvas}{Sk\+Canvas}} draw member functions. \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} flexibility as a pixel container limits some optimizations available to the target platform.

If pixel array is primarily read-\/only, use \mbox{\hyperlink{class_sk_image}{Sk\+Image}} for better performance. If pixel array is primarily written to, use \mbox{\hyperlink{class_sk_surface}{Sk\+Surface}} for better performance.

Declaring \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} const prevents altering \mbox{\hyperlink{struct_sk_image_info}{Sk\+Image\+Info}}\+: the \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} height, width, and so on cannot change. It does not affect \mbox{\hyperlink{class_sk_pixel_ref}{Sk\+Pixel\+Ref}}\+: a caller may write its pixels. Declaring \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} const affects \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} configuration, not its contents.

\mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} is not thread safe. Each thread must have its own copy of \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} fields, although threads may share the underlying pixel array.

Copies a \mbox{\hyperlink{struct_sk_rect}{Sk\+Rect}} of pixels from src. Copy starts at (dstX, dstY), and does not exceed (src.\+width(), src.\+height()).

src specifies width, height, Sk\+Color\+Type, Sk\+Alpha\+Type, \mbox{\hyperlink{class_sk_color_space}{Sk\+Color\+Space}}, pixel storage, and row bytes of source. src.\+row\+Bytes() specifics the gap from one source row to the next. Returns true if pixels are copied. Returns false if\+:
\begin{DoxyItemize}
\item src pixel storage equals nullptr
\item src.\+row\+Bytes is less than \mbox{\hyperlink{struct_sk_image_info_a48bbcae20c25171cf36a719af8d8599d}{Sk\+Image\+Info\+::min\+Row\+Bytes()}}
\item \mbox{\hyperlink{class_sk_pixel_ref}{Sk\+Pixel\+Ref}} is nullptr
\end{DoxyItemize}

Pixels are copied only if pixel conversion is possible. If \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} color\+Type() is k\+Gray\+\_\+8\+\_\+\+Sk\+Color\+Type, or k\+Alpha\+\_\+8\+\_\+\+Sk\+Color\+Type; src Sk\+Color\+Type must match. If \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} color\+Type() is k\+Gray\+\_\+8\+\_\+\+Sk\+Color\+Type, src \mbox{\hyperlink{class_sk_color_space}{Sk\+Color\+Space}} must match. If \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} alpha\+Type() is k\+Opaque\+\_\+\+Sk\+Alpha\+Type, src Sk\+Alpha\+Type must match. If \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} \mbox{\hyperlink{class_sk_bitmap_a0b0e607a828dbece8093d78407a085ec}{color\+Space()}} is nullptr, src \mbox{\hyperlink{class_sk_color_space}{Sk\+Color\+Space}} must match. Returns false if pixel conversion is not possible.

dstX and dstY may be negative to copy only top or left of source. Returns false if \mbox{\hyperlink{class_sk_bitmap_a5154594e432462e242daa8a7515bd6d9}{width()}} or \mbox{\hyperlink{class_sk_bitmap_ad9cc641616d2cf9e4e915c1bfd390453}{height()}} is zero or negative.
