Returns the \mbox{\hyperlink{class_sk_image}{Sk\+Image}} for a given frame.

If the image asset is static, \mbox{\hyperlink{classskresources_1_1_multi_frame_image_asset_ad06e0c4aea0323621f8098b0172fd538}{get\+Frame()}} is only called once, at animation load time. Otherwise, this gets invoked every time the animation time is adjusted (on every seek).

Embedders should cache and serve the same \mbox{\hyperlink{class_sk_image}{Sk\+Image}} whenever possible, for efficiency.

By default, images are decoded on-\/the-\/fly, at rasterization time. Large images may cause jank as decoding is expensive (and can thrash internal caches).

Pass $\vert$predecode$\vert$ true to force-\/decode all images upfront, at the cost of potentially more R\+AM and slower animation build times.
