\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}} provides a utility to pair \mbox{\hyperlink{struct_sk_image_info}{Sk\+Image\+Info}} with pixels and row bytes. \mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}} is a low level class which provides convenience functions to access raster destinations. \mbox{\hyperlink{class_sk_canvas}{Sk\+Canvas}} can not draw \mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}}, nor does \mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}} provide a direct drawing destination.

Use \mbox{\hyperlink{class_sk_bitmap}{Sk\+Bitmap}} to draw pixels referenced by \mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}}; use \mbox{\hyperlink{class_sk_surface}{Sk\+Surface}} to draw into pixels referenced by \mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}}.

\mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}} does not try to manage the lifetime of the pixel memory. Use \mbox{\hyperlink{class_sk_pixel_ref}{Sk\+Pixel\+Ref}} to manage pixel memory; \mbox{\hyperlink{class_sk_pixel_ref}{Sk\+Pixel\+Ref}} is safe across threads.

{\footnotesize\ttfamily Sk\+Pixmap\+::\+Sk\+Pixmap (\begin{DoxyParamCaption}\item[{const \mbox{\hyperlink{struct_sk_image_info}{Sk\+Image\+Info}} \&}]{info,  }\item[{const void $\ast$}]{addr,  }\item[{size\+\_\+t}]{row\+Bytes }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}

Creates \mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}} from info width, height, Sk\+Alpha\+Type, and Sk\+Color\+Type. addr points to pixels, or nullptr. row\+Bytes should be info.\+width() times info.\+bytes\+Per\+Pixel(), or larger.

No parameter checking is performed; it is up to the caller to ensure that addr and row\+Bytes agree with info.

The memory lifetime of pixels is managed by the caller. When \mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}} goes out of scope, addr is unaffected.

\mbox{\hyperlink{class_sk_pixmap}{Sk\+Pixmap}} may be later modified by \mbox{\hyperlink{class_sk_pixmap_af77d3d9a0d150069f27fe5a6278a03fb}{reset()}} to change its size, pixel type, or storage.
