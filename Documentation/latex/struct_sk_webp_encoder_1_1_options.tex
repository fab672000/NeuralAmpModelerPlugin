{\footnotesize\ttfamily Compression Sk\+Webp\+Encoder\+::\+Options\+::f\+Compression = Compression\+::k\+Lossy}

$\vert$f\+Compression$\vert$ determines whether we will use webp lossy or lossless compression.

$\vert$f\+Quality$\vert$ must be in \mbox{[}0.\+0f, 100.\+0f\mbox{]}. If $\vert$f\+Compression$\vert$ is k\+Lossy, $\vert$f\+Quality$\vert$ corresponds to the visual quality of the encoding. Decreasing the quality will result in a smaller encoded image. If $\vert$f\+Compression$\vert$ is k\+Lossless, $\vert$f\+Quality$\vert$ corresponds to the amount of effort put into the encoding. Lower values will compress faster into larger files, while larger values will compress slower into smaller files.

This scheme is designed to match the libwebp A\+PI.
