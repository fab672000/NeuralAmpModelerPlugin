faust2max6/faust2msp transform a \mbox{\hyperlink{struct_faust}{Faust}} \mbox{\hyperlink{class_d_s_p}{D\+SP}} program into a compiled Max/\+M\+SP external, and a ready-\/to-\/use patch to load and use it. Polyphonic and M\+I\+DI controllable synthesisers can be created. Note that faust2msp is the old version compiling 32 bits external for Max5, and faust2max6 compiles 64 bits or 64/32 bits externals for Max6 and later versions, and should be preferably used.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_e139210c7459bd8b712221dd9976dc81_autotoc_md294}{}\doxysection{How to use}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_e139210c7459bd8b712221dd9976dc81_autotoc_md294}
faust2max6 is used with the following parameters\+:

{\ttfamily faust2max6 \mbox{[}-\/opt native$\vert$generic\mbox{]} \mbox{[}-\/native\mbox{]} \mbox{[}-\/nvoices $<$num$>$\mbox{]} \mbox{[}-\/effect $<$effect.\+dsp$>$\mbox{]} \mbox{[}-\/midi\mbox{]} \mbox{[}-\/osc\mbox{]} \mbox{[}-\/us $<$factor$>$\mbox{]} \mbox{[}-\/ds $<$factor$>$\mbox{]} \mbox{[}-\/filter $<$filter$>$\mbox{]} \mbox{[}-\/universal\mbox{]} \mbox{[}-\/nopatch\mbox{]} \mbox{[}-\/soundfile/-\/soundfile-\/static\mbox{]} \mbox{[}additional \mbox{\hyperlink{struct_faust}{Faust}} options (-\/vec -\/vs 8...)\mbox{]} $<$file.\+dsp$>$}

By default it will create {\itshape file$\sim$.mxo} external along with a {\itshape file.\+maxpat} patch file and a {\itshape ui.\+js} helper file, that will load the external and automatically create a User Interface (with sliders, buttons...) ready to control it. To be fully functional, the object still has to be connected to audio inputs/outputs or other elements in the patch. {\bfseries{Double-\/click}} on the object allow to display its controls with their {\bfseries{label}}, {\bfseries{range}} and {\bfseries{complete path}}. Note that {\itshape -\/double} compilation mode is used by default in faust2max6.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_e139210c7459bd8b712221dd9976dc81_autotoc_md295}{}\doxysection{Options}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_e139210c7459bd8b712221dd9976dc81_autotoc_md295}
The following options are available\+:


\begin{DoxyItemize}
\item {\ttfamily -\/opt native} \+: to activate the best compilation options for the native C\+PU
\item {\ttfamily -\/opt generic} \+: to activate the best compilation options for a generic C\+PU
\item {\ttfamily -\/native} to compile for the native C\+PU (otherwise the \textquotesingle{}generic\textquotesingle{} mode is used by default)
\item {\ttfamily -\/nvoices $<$num$>$} \+: to produce a polyphonic self-\/contained \mbox{\hyperlink{class_d_s_p}{D\+SP}} with $<$num$>$ voices, ready to be used with M\+I\+DI
\item {\ttfamily -\/effect $<$effect.\+dsp$>$} \+: to produce a polyphonic \mbox{\hyperlink{class_d_s_p}{D\+SP}} connected to a global output effect, ready to be used with M\+I\+DI
\item {\ttfamily -\/midi} \+: to activate M\+I\+DI control
\item {\ttfamily -\/osc} \+: to activate O\+SC control
\item {\ttfamily -\/us $<$factor$>$} \+: upsample the \mbox{\hyperlink{class_d_s_p}{D\+SP}} by a factor
\item {\ttfamily -\/ds $<$factor$>$} \+: downsample the \mbox{\hyperlink{class_d_s_p}{D\+SP}} by a factor
\item {\ttfamily -\/filter $<$filter$>$} \+: use a filter for upsampling or downsampling \mbox{[}0..4\mbox{]}
\item {\ttfamily -\/universal} \+: to generate a 64/32 bits external
\item {\ttfamily -\/nopatch} \+: to deactivate patch generation
\item {\ttfamily -\/soundfile} \+: when compiling \mbox{\hyperlink{class_d_s_p}{D\+SP}} using \textquotesingle{}soundfile\textquotesingle{} primitive, to add needed resources
\item {\ttfamily -\/soundfile-\/static} when compiling \mbox{\hyperlink{class_d_s_p}{D\+SP}} using \textquotesingle{}soundfile\textquotesingle{} primitive, to add needed resources in static mode
\item {\ttfamily -\/help or -\/h} \+: shows the different options
\end{DoxyItemize}

The {\itshape faust2msp} tools has to be used to create old Max5 compatible externals, the {\itshape faust2max6} tools has to be used starting with Max6.\hypertarget{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_e139210c7459bd8b712221dd9976dc81_autotoc_md296}{}\doxysection{Configuration}\label{md__c_1__users_fab_src__github_branches__neural_amp_modeler_plugin_i_plug2__dependencies__build_e139210c7459bd8b712221dd9976dc81_autotoc_md296}
If you plan to use faust2max6/faust2msp on your own machine, you will have to\+:


\begin{DoxyItemize}
\item install \href{https://cycling74.com/downloads/sdk}{\texttt{ Max/\+M\+SP S\+DK}}
\item edit the {\ttfamily faust/tools/faust2appls/faustpath} script to properly setup the \$\+M\+A\+X\+S\+DK variable, then re-\/install the scripts again using {\ttfamily sudo make install}
\item alternatively you can locally change the \$\+M\+A\+X\+S\+DK variable in a terminal using {\ttfamily export M\+A\+X\+S\+DK=/your/alternate/path}, then use faust2msp/faust2max6 in this terminal. 
\end{DoxyItemize}
