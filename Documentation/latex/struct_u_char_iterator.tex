\doxysubsection{Detailed Description}
C A\+PI for code unit iteration. This can be used as a C wrapper around Character\+Iterator, Replaceable, or implemented using simple strings, etc.

There are two roles for using \mbox{\hyperlink{struct_u_char_iterator}{U\+Char\+Iterator}}\+:

A \char`\"{}provider\char`\"{} sets the necessary function pointers and controls the \char`\"{}protected\char`\"{} fields of the \mbox{\hyperlink{struct_u_char_iterator}{U\+Char\+Iterator}} structure. A \char`\"{}provider\char`\"{} passes a \mbox{\hyperlink{struct_u_char_iterator}{U\+Char\+Iterator}} into C A\+P\+Is that need a \mbox{\hyperlink{struct_u_char_iterator}{U\+Char\+Iterator}} as an abstract, flexible string interface.

Implementations of such C A\+P\+Is are \char`\"{}callers\char`\"{} of \mbox{\hyperlink{struct_u_char_iterator}{U\+Char\+Iterator}} functions; they only use the \char`\"{}public\char`\"{} function pointers and never access the \char`\"{}protected\char`\"{} fields directly.

The \mbox{\hyperlink{struct_u_char_iterator_ac247dee1b3e3e6fec6688f9dcb5bc1c6}{current()}} and \mbox{\hyperlink{struct_u_char_iterator_aae26243728a62e6a13ed2008972afb50}{next()}} functions only check the current index against the limit, and \mbox{\hyperlink{struct_u_char_iterator_a65cd2a527c6abe69a0f37a368ce51cfb}{previous()}} only checks the current index against the start, to see if the iterator already reached the end of the iteration range.

The assumption -\/ in all iterators -\/ is that the index is moved via the A\+PI, which means it won\textquotesingle{}t go out of bounds, or the index is modified by user code that knows enough about the iterator implementation to set valid index values.

\mbox{\hyperlink{struct_u_char_iterator}{U\+Char\+Iterator}} functions return code unit values 0..0xffff, or U\+\_\+\+S\+E\+N\+T\+I\+N\+EL if the iteration bounds are reached.
