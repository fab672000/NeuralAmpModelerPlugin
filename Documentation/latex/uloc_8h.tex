\doxysubsection{Detailed Description}
C A\+PI\+: Locale. 

\doxysubsubsection*{U\+Loc C A\+PI for Locale }

A {\ttfamily Locale} represents a specific geographical, political, or cultural region. An operation that requires a {\ttfamily Locale} to perform its task is called {\itshape locale-\/sensitive} and uses the {\ttfamily Locale} to tailor information for the user. For example, displaying a number is a locale-\/sensitive operation--the number should be formatted according to the customs/conventions of the user\textquotesingle{}s native country, region, or culture. In the C A\+P\+Is, a locales is simply a const char string.

You create a {\ttfamily Locale} with one of the three options listed below. Each of the component is separated by \textquotesingle{}\+\_\+\textquotesingle{} in the locale string.  
\begin{DoxyPre}

\begin{DoxyCode}{0}
\DoxyCodeLine{newLanguage}
\DoxyCodeLine{}
\DoxyCodeLine{newLanguage + newCountry}
\DoxyCodeLine{}
\DoxyCodeLine{newLanguage + newCountry + newVariant}
\end{DoxyCode}

\end{DoxyPre}
  The first option is a valid {\bfseries{I\+SO Language Code.}} These codes are the lower-\/case two-\/letter codes as defined by I\+S\+O-\/639. You can find a full list of these codes at a number of sites, such as\+: ~\newline
\href{http://www.ics.uci.edu/pub/ietf/http/related/iso639.txt}{\texttt{ http\+://www.\+ics.\+uci.\+edu/pub/ietf/http/related/iso639.\+txt}}

The second option includes an additional {\bfseries{I\+SO Country Code.}} These codes are the upper-\/case two-\/letter codes as defined by I\+S\+O-\/3166. You can find a full list of these codes at a number of sites, such as\+: ~\newline
\href{http://www.chemie.fu-berlin.de/diverse/doc/ISO_3166.html}{\texttt{ http\+://www.\+chemie.\+fu-\/berlin.\+de/diverse/doc/\+I\+S\+O\+\_\+3166.\+html}}

The third option requires another additional information--the {\bfseries{Variant.}} The Variant codes are vendor and browser-\/specific. For example, use W\+IN for Windows, M\+AC for Macintosh, and P\+O\+S\+IX for P\+O\+S\+IX. Where there are two variants, separate them with an underscore, and put the most important one first. For example, a Traditional Spanish collation might be referenced, with \char`\"{}\+E\+S\char`\"{}, \char`\"{}\+E\+S\char`\"{}, \char`\"{}\+Traditional\+\_\+\+W\+I\+N\char`\"{}.

Because a {\ttfamily Locale} is just an identifier for a region, no validity check is performed when you specify a {\ttfamily Locale}. If you want to see whether particular resources are available for the {\ttfamily Locale} you asked for, you must query those resources. For example, ask the {\ttfamily U\+Number\+Format} for the locales it supports using its {\ttfamily get\+Available} method. ~\newline
{\bfseries{Note\+:}} When you ask for a resource for a particular locale, you get back the best available match, not necessarily precisely what you asked for. For more information, look at {\ttfamily U\+Resource\+Bundle}.

The {\ttfamily Locale} provides a number of convenient constants that you can use to specify the commonly used locales. For example, the following refers to a locale for the United States\+:

Once you\textquotesingle{}ve specified a locale you can query it for information about itself. Use {\ttfamily uloc\+\_\+get\+Country} to get the I\+SO Country Code and {\ttfamily uloc\+\_\+get\+Language} to get the I\+SO Language Code. You can use {\ttfamily uloc\+\_\+get\+Display\+Country} to get the name of the country suitable for displaying to the user. Similarly, you can use {\ttfamily uloc\+\_\+get\+Display\+Language} to get the name of the language suitable for displaying to the user. Interestingly, the {\ttfamily uloc\+\_\+get\+Display\+X\+XX} methods are themselves locale-\/sensitive and have two versions\+: one that uses the default locale and one that takes a locale as an argument and displays the name or country in a language appropriate to that locale.

The I\+CU provides a number of services that perform locale-\/sensitive operations. For example, the {\ttfamily unum\+\_\+xxx} functions format numbers, currency, or percentages in a locale-\/sensitive manner.

Concerning P\+O\+S\+I\+X/\+R\+F\+C1766 Locale I\+Ds, the get\+Language/get\+Country/get\+Variant/get\+Name functions do understand the P\+O\+S\+IX type form of language\+\_\+\+C\+O\+U\+N\+T\+R\+Y.\+E\+N\+C\+O\+D\+I\+NG@V\+A\+R\+I\+A\+NT and if there is not an I\+C\+U-\/stype variant, \mbox{\hyperlink{uloc_8h_ac6949841ca8e8f06fa8cbf1d0a2e3378}{uloc\+\_\+get\+Variant()}} for example will return the one listed after the @at sign. As well, the hyphen \char`\"{}-\/\char`\"{} is recognized as a country/variant separator similarly to R\+F\+C1766. So for example, \char`\"{}en-\/us\char`\"{} will be interpreted as en\+\_\+\+US. ~\newline
 As a result, \mbox{\hyperlink{uloc_8h_a1afb3b0aa043993207a6950233cf26a8}{uloc\+\_\+get\+Name()}} is far from a no-\/op, and will have the effect of converting P\+O\+S\+I\+X/\+R\+F\+C1766 I\+Ds into I\+CU form, although it does N\+OT map any of the actual codes (i.\+e. russian-\/$>$ru) in any way. Applications should call \mbox{\hyperlink{uloc_8h_a1afb3b0aa043993207a6950233cf26a8}{uloc\+\_\+get\+Name()}} at the point where a locale ID is coming from an external source (user entry, OS, web browser) and pass the resulting string to other I\+CU functions. For example, don\textquotesingle{}t use de-\/de@E\+U\+RO as an argument to resourcebundle.
