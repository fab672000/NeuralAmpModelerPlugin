\doxysubsection{Detailed Description}
class Bidiagonal Divide and Conquer S\+VD 


\begin{DoxyTemplParams}{Template Parameters}
{\em Matrix\+Type\+\_\+} & the type of the matrix of which we are computing the S\+VD decomposition\\
\hline
{\em Options\+\_\+} & this optional parameter allows one to specify options for computing unitaries {\itshape U} and {\itshape V}. Possible values are \mbox{\hyperlink{group__enums_ggae3e239fb70022eb8747994cf5d68b4a9af8c742a1aa87773e165eae406c9ccaf8}{Compute\+ThinU}}, \mbox{\hyperlink{group__enums_ggae3e239fb70022eb8747994cf5d68b4a9a1055e53fa95c8ae04a07ebb72cfafd95}{Compute\+ThinV}}, \mbox{\hyperlink{group__enums_ggae3e239fb70022eb8747994cf5d68b4a9a2b4f91ca5859a4159dbfe8090043817f}{Compute\+FullU}}, \mbox{\hyperlink{group__enums_ggae3e239fb70022eb8747994cf5d68b4a9a52c6f7e80bbf9a42297c88f700245b51}{Compute\+FullV}}, and \mbox{\hyperlink{group__enums_gga46eba0d5c621f590b8cf1b53af31d56eae5c5f7ab41066cf3f9f976c28a0cf918}{Disable\+Q\+R\+Decomposition}}. It is not possible to request both the thin and full version of {\itshape U} or {\itshape V}. By default, unitaries are not computed. \mbox{\hyperlink{class_eigen_1_1_b_d_c_s_v_d}{B\+D\+C\+S\+VD}} uses R-\/\+Bidiagonalization to improve performance on tall and wide matrices. For backwards compatility, the option \mbox{\hyperlink{group__enums_gga46eba0d5c621f590b8cf1b53af31d56eae5c5f7ab41066cf3f9f976c28a0cf918}{Disable\+Q\+R\+Decomposition}} can be used to disable this optimization.\\
\hline
\end{DoxyTemplParams}
This class first reduces the input matrix to bi-\/diagonal form using class Upper\+Bidiagonalization, and then performs a divide-\/and-\/conquer diagonalization. Small blocks are diagonalized using class \mbox{\hyperlink{class_eigen_1_1_jacobi_s_v_d}{Jacobi\+S\+VD}}. You can control the switching size with the set\+Switch\+Size() method, default is 16. For small matrice ($<$16), it is thus preferable to directly use \mbox{\hyperlink{class_eigen_1_1_jacobi_s_v_d}{Jacobi\+S\+VD}}. For larger ones, \mbox{\hyperlink{class_eigen_1_1_b_d_c_s_v_d}{B\+D\+C\+S\+VD}} is highly recommended and can several order of magnitude faster.

\begin{DoxyWarning}{Warning}
this algorithm is unlikely to provide accurate result when compiled with unsafe math optimizations. For instance, this concerns Intel\textquotesingle{}s compiler (I\+CC), which performs such optimization by default unless you compile with the {\ttfamily -\/fp-\/model} {\ttfamily precise} option. Likewise, the {\ttfamily -\/ffast-\/math} option of G\+CC or clang will significantly degrade the accuracy.
\end{DoxyWarning}
