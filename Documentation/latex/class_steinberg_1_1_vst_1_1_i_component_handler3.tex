\doxysubsection{Detailed Description}
Extended host callback interface \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_component_handler3}{Vst\+::\+I\+Component\+Handler3}} for an edit controller.


\begin{DoxyItemize}
\item \mbox{[}host imp\mbox{]}
\item \mbox{[}extends \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_component_handler}{I\+Component\+Handler}}\mbox{]}
\item \mbox{[}released\+: 3.\+5.\+0\mbox{]}
\item \mbox{[}optional\mbox{]}
\end{DoxyItemize}

A plug-\/in can ask the host to create a context menu for a given exported parameter ID or a generic context menu.~\newline
 The host may pre-\/fill this context menu with specific items regarding the parameter ID like \char`\"{}\+Show automation for parameter\char`\"{}, \char`\"{}\+M\+I\+D\+I learn\char`\"{} etc...~\newline
 The plug-\/in can use the context menu in two ways \+:
\begin{DoxyItemize}
\item add its own items to the menu via the \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_context_menu}{I\+Context\+Menu}} interface and call \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_context_menu_a740811258f789a6da82b3155f7191dac}{I\+Context\+Menu\+::popup}}(..) to create the pop-\/up. See the \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_component_handler3_IContextMenuExample}{Examples}}.
\item extract the host menu items and add them to a context menu created by the plug-\/in.
\end{DoxyItemize}

{\bfseries{Note\+:}} You can and should use this even if you do not add your own items to the menu as this is considered to be a big user value.
